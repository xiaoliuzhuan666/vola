# Vola 核心工作台 UI 方向

日期：2026-06-18

## 核心不变

Vola 的日常界面只服务四件事：

1. 管理个人 Skill。
2. 管理个人 MCP。
3. 团队共享 Skill 和 MCP。
4. 管理项目 Markdown，并把真实路径复制给 Codex。

账号、导入、记忆、优化建议、CLI、开发者访问仍然需要保留，但不应该挤占主入口。

## 参考产品观察

- Linear 的侧边栏强调个性化、收藏和少量高频入口，复杂内容通过 workspace 和 project 层级进入。
- Slack 的信息架构把频道、列表、应用和更多入口分组，主侧边栏不承载所有功能。
- Notion Wiki / Confluence 更适合知识管理：搜索、页面、数据库/表格、模板比深层卡片更高效。
- GitHub Projects 证明同一批信息可以有 table、board、roadmap 多种视图，列表和表格适合高密度管理。
- Raycast Teams 聚焦共享扩展、Quicklinks、Snippets，团队价值来自可复用资产，而不是大而全的后台。

## Vola 的界面原则

- 左侧只放主任务：Codex Console、Skill、MCP、团队、项目文档。
- “更多”放低频能力：导入、记忆、优化建议、云端账号、备份、CLI、开发者访问、系统设置。
- Skill、MCP、团队资产优先使用列表、表格、分栏和筛选，不再继续堆大卡片。
- 需要创建或配置时用弹窗或右侧面板，不让表单常驻在主页面。
- 项目 Markdown 索引不是自动真相库，只是帮人和 Codex 找到应该读取的本地文件。
- Vola 的“自进化”来自人确认后的资料沉淀：Codex 读路径、完成任务、用户确认，再写回 Skill、MCP、项目文档或团队资产。

## 当前落地

- 左侧导航简化为核心入口。
- 团队页改为 tab：概览、资产、同步、文件。
- 团队资产页主区域聚焦 Skill/MCP、同步、审查状态，其他资料放到目录侧栏或次级折叠区。
- Codex Console 增加“项目文档”入口，可以扫描本机 Markdown、按类型分组、推荐选择、复制真实路径和提示词。

## 暂不做

- 不做自动改写用户项目文档的自治流程。
- 不把所有 Markdown 全文一次性塞给 Codex。
- 不把团队页做成后台管理大杂烩。
- 不为了显得智能而增加无关统计卡片。
